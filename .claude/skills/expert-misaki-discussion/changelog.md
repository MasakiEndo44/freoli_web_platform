## 2026-05-05 (2): Agent Team Mode が一度も起動しない問題を修正

- **きっかけ**: 直後フィードバック — `team` フラグを渡しても Classic Mode が起動し続けた
- **ギャップ**: `team` などのフラグを渡しても Agent Team Mode が起動しない。Classic Mode 以外が起動したことが一度もない
- **根本原因**: 指示不足 × 文脈理解不足（2重）
  - "ARGUMENTS" という語が Cursor IDE に存在しない概念のため、AIがどこを見ればよいか判断できていなかった
  - モード選択が `>` 引用ブロック（補足注意書き）として書かれており、必須実行フローとして処理されにくい構造だった
- **変更内容**:
  - 「実行モードの選択」セクションを「⚡ Step 0：実行モードの判定」に格上げ。Step 1 より前に実行する必須ステップとして明示
  - "ARGUMENTS" を「ユーザーのメッセージ（このスキルを呼び出した入力テキスト）」に変更し、Cursor での参照対象を明確化
  - フラグ検出時・トリガーワード一致時に「今すぐ宣言して進む」という即時アクションを義務付け
  - Step 2 冒頭に「Classic Mode のみ進む」ガード、Step 2-B 冒頭に「Agent Team Mode のみ進む」ガードを追加

## 2026-05-05: Agent Team Mode のモード判定を修正

- **きっかけ**: 直後フィードバック（skill-improverによる）
- **ギャップ**: 「本気の議論で」「本気で議論して」などを指定してもClassic Modeで実行される。`/expert-misaki-discussion team` のような引数フラグも認識されない
- **根本原因**: 指示不足 — トリガーワードはリストされていたが「検出したらStep 2-Bに進め」というif-then命令がなく、Classic Modeへの重力が勝っていた。引数フラグの仕組みも未定義
- **変更内容**:
  - モード選択の判定ブロックを引数フラグ優先 → トリガーワード → Classic Modeのif-then構造に再設計
  - 引数フラグ追加（`team` / `--team` / `-t` / `agents` / `deep` / `ガチ` / `本気`）
  - トリガーワードに「本気で議論して」「本気モードで」「ガチ議論」「マルチエージェントで」を追加
  - 「最初に必ず判定する（スキップ禁止）」の強調でモデルの読み飛ばし防止
