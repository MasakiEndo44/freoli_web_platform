# goal-grill — Web Escalation Policy

goal-grill 内での Web 検索ゲートの発動条件・手順・安全基準。  
（requirements-grill の WEB_ESCALATION.md を 3 層用に簡略化したもの）

---

## 発動条件

以下のいずれかで `web_trigger = true` になる。

| 条件 | 説明 |
|---|---|
| A | 問いの推奨回答が「最終手段」文言（"まだ情報が不足しています"）で終わる |
| B | 推奨回答に「年」「バージョン」「API」「SDK」「モデル名」「価格」「仕様」が含まれる |
| C | ユーザーが「最新は？」「調べて」「今どうなっている？」と言った |
| D | E 層で受け入れ基準に特定ツール・ライブラリのバージョンが関係する |

以下の場合は `web_trigger = false`（抑制）:

- ユーザーが同じ層で既に 2 回 "n" を選択した（その層はWeb検索を抑制）
- V 層の価値観・思想に関する問い（事実確認ではなく判断の問い）

---

## 発動時のプロンプト

```
🔎 Web で最新確認しますか？
   対象: <何を確認するか 1 行>
   クエリ案: "<search term>"

   [Y: 検索する / n: 推論デフォルトで進む / skip: この問いを [X] 保留]
```

---

## 検索後の処理

```
Y が選ばれた場合:
  1. WebSearch または WebFetch を実行
  2. 結果から 2〜3 行のサマリーを抽出
  3. 出典（ドメイン or URL）を記録
  4. "この情報を推奨回答に採用しますか？ [Y/n]" を表示
  5. Y → 推奨回答を更新、ラベルを [W: source] にする
     n → 推論デフォルトで進む

ツールが利用不可の場合:
  → エラーを表示せず、推論デフォルトで縮退
  → セッションノートに "手動確認推奨: <対象>" を記録
```

---

## 安全基準

- 検索結果をそのまま推奨回答にしない。2〜3 行のサマリーで要約する
- 出典を必ず記録する（後で人間が確認できるように）
- 個人情報・機密情報を含む可能性のあるクエリは実行しない
- 検索結果の採用は必ず人間が承認する（[W: source] ラベルで追跡）
