# Goal Grill — 3 Layer Definitions

goal-grill が使う 3 層の問い一覧、推奨デフォルト値、ラベル判定ガイド。

---

## Layer V: Vision（なぜ作るか）

### 目的

このタスク/機能が解決する問題と、誰のために作るかを明確にする。

### 問い一覧

| No | 問い | デフォルト（情報不足時） |
|---|---|---|
| V-1 | このタスク/機能が解決する**一番大きな問題**は何ですか？ | "まだ情報が不足しています。ご意見をお聞かせください" |
| V-2 | 主な**ユーザー/受益者**は誰ですか？ | 「このrepoの主要ユーザー」（HARNESS_DECISIONS から推論） |
| V-3 | このタスク/機能がなければ、ユーザーは今どうやって解決していますか？（代替手段） | "手動作業、または未解決" |
| V-4 | 6ヶ月後、このタスク/機能が成功していたらどんな状態ですか？（1文） | "まだ情報が不足しています。ご意見をお聞かせください" |

最小問数: **V-1 と V-4 は必須**。V-2 / V-3 は推定可能なら推定して確認する。

---

## Layer O: Outcome（何が変わるか）

### 目的

完了後に測定できる変化（指標・対象・期間）を定める。

### 問い一覧

| No | 問い | デフォルト（情報不足時） |
|---|---|---|
| O-1 | このタスク/機能の完了後、**何が変わったと言えますか**？（動詞+対象+程度） | "まだ情報が不足しています。ご意見をお聞かせください" |
| O-2 | その変化を**どうやって測りますか**？（定量 or 定性の指標） | "ユーザーインタビューまたはアクセスログ" |
| O-3 | 成功の**期限**はありますか？（ない場合は "継続的に" でも可） | "継続的に" |
| O-4 | 成功/失敗を判断する**最低ライン**（これを下回ったら失敗と言う閾値）はありますか？ | [X] 保留推奨（後でE層で詳細化） |

最小問数: **O-1 と O-2 は必須**。

---

## Layer E: Eval（どう確認するか）

### 目的

このタスク/機能の「完了」と「品質合格」を判定するための受け入れ基準を定める。  
EARS 記法（When/While/If/Where/The system shall）を推奨するが、平文でも可。

### 問い一覧

| No | 問い | デフォルト（情報不足時） |
|---|---|---|
| E-1 | このタスク/機能が「完成した」と言えるための**最低限の動作**は何ですか？（1〜3件） | "まだ情報が不足しています。ご意見をお聞かせください" |
| E-2 | 「壊れている」と判断する**失敗ケース**を1つ挙げてください。 | "まだ情報が不足しています。ご意見をお聞かせください" |
| E-3 | この受け入れ基準を**誰がどうやって確認**しますか？（人間レビュー / 自動テスト / 両方） | "人間レビュー" |
| E-4 | 今後この機能が劣化したとき、**検出できるsensorはありますか**？（テスト/モニタリング/定期確認） | [X] 保留推奨（HARNESS_HEALTH.md に記録） |

最小問数: **E-1 と E-2 は必須**。

---

## 書き出しフォーマット

`docs/output/goals/<slug>.md` に以下のフォーマットで書き出す。

```markdown
---
task: <タスク/機能名>
created: <YYYY-MM-DD>
status: draft
---

# Goal: <タスク/機能名>

## Vision

**問題**: <V-1>
**ユーザー**: <V-2>
**代替手段**: <V-3>
**6ヶ月後の成功像**: <V-4>

## Outcome

**何が変わるか**: <O-1>
**測定方法**: <O-2>
**期限**: <O-3>
**最低ライン**: <O-4 or 未決>

## Eval criteria

- [ ] <E-1-a: 最低限の動作1>
- [ ] <E-1-b: 最低限の動作2（あれば）>
- [ ] <E-2: 失敗ケース（これが起きたら壊れている）>

**確認方法**: <E-3>
**劣化検出**: <E-4 or 未決 — HARNESS_HEALTH.md に記録>

## Open questions

- [X] <未決項目があれば列挙>
```
