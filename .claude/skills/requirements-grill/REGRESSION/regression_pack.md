# requirements-grill 回帰テストパック

このファイルは `requirements-grill` スキルの回帰テスト仕様です。  
skill-regression-pack スキルで整備し、スキル改善のたびに実行します。

---

## Positive Cases（正常発火・正常動作）

### P-01: NEW モード — 自然言語トリガー

**入力**: 「[プロダクト名] の要件を詰めたい」  
**期待動作**:
- `requirements-grill` が発火する（他のスキルが発火しない）
- OPENING LINE が表示される（モード: NEW）
- Q-INIT-1（プロダクト選択）が聞かれる
- 推奨回答が添えられている

**合格条件**:
- [ ] OPENING LINE に「Requirements Grill を起動しました」が含まれる
- [ ] Q-INIT-1 が最初の問いとして表示される
- [ ] 推奨回答が提示される

---

### P-02: NEW モード — 別トリガー表現

**入力**: 「vision を固めたい」  
**期待動作**: P-01 と同じ。current_layer が V に設定される

**合格条件**: P-01 と同様

---

### P-03: RESUME モード — セッションファイル添付

**入力**: `docs/sandbox/<member>/grill-sessions/<session-file>.md` を添付して「続きをやりたい」  
**期待動作**:
- モード: RESUME が表示される
- セッションファイルの `## State` を読んで `next_action` から再開する
- `## Open Questions` から未回答のものを最初の問いとして出す

**合格条件**:
- [ ] OPENING LINE に「モード: RESUME」が含まれる
- [ ] セッションファイルの `current_layer` を正しく読み取る
- [ ] Open Questions の最初の項目から再開する

---

### P-04: REVIEW モード — 既存ファイル更新

**入力**: `docs/requirements/<product>/capabilities/CAP-001_<slug>.md` を添付して「この capability を更新したい」  
**期待動作**:
- モード: REVIEW が表示される
- 対象ファイルの既存フロントマターを読んで値を確認する
- QUESTIONS_C.md をロードして差分 grill を開始する

**合格条件**:
- [ ] OPENING LINE に「モード: REVIEW」が含まれる
- [ ] 既存の `one_line_thesis` を引用して「これは正しいですか？」と確認する

---

### P-05: WRITEOUT_ONLY モード

**入力**: セッションファイルを添付して「書き出したい」  
**期待動作**:
- モード: WRITEOUT_ONLY
- セッションの `## Layer Progress` で `ready_to_writeout: true` を探す
- WRITEOUT GATE (Preview → Label Audit → Approval) を実行する

**合格条件**:
- [ ] OPENING LINE に「モード: WRITEOUT_ONLY」が含まれる
- [ ] 「承認」の前にはファイルを書かない

---

### P-06: 1問1推奨回答のフォーマット

**任意の問いへの応答フォーマット確認**  
**期待動作**:
```
--- [V-1] ---
Q: このプロダクトは「誰の・何の痛みを」解きますか？
推奨: [20_products/<product>/problem.md から引用した内容]
(Enter で推奨採用 / 別の回答を入力 / 「後で」で [X] 保留)
```

**合格条件**:
- [ ] 問い番号が `[層-番号]` 形式で表示される
- [ ] 「推奨:」が必ず含まれる
- [ ] 操作オプションが表示される

---

### P-07: ラベリングの正確性

**入力**: Q に対してユーザーが「はい、それで」と答える  
**期待動作**: `[A]` ラベルで記録される  

**入力**: ユーザーが具体的な内容を語る  
**期待動作**: `[U]` ラベルで記録される  

**入力**: 「後で」「保留」と答える  
**期待動作**: `[X]` ラベルで記録され、Open Questions に残る

---

### P-08: WriteoutGate の3段階

**期待動作**:
1. PREVIEW で Frontmatter 全フィールドが表示される
2. LABEL AUDIT で `[I]` ラベルの値が強調される
3. ユーザーが「承認」と言うまでファイルが書かれない

**合格条件**:
- [ ] 「承認」の前に Write ツールが呼ばれない
- [ ] LABEL AUDIT で `[I]` フィールドに「⚠️」が付く

---

## Negative Cases（誤発火しない / 誤動作しない）

### N-01: 雑談では発火しない

**入力**: 「今日の天気は？」「今週の予定を教えて」  
**期待動作**: `requirements-grill` は発火しない。他のスキルか通常回答が返る

**合格条件**:
- [ ] OPENING LINE が表示されない

---

### N-02: ops タスク依頼では発火しない

**入力**: 「タスクを更新して」「会議メモをコンパイルして」  
**期待動作**: `requirements-grill` は発火しない（compile-meeting スキル等が優先）

---

### N-03: 本文を創作しない（guardrails）

**状況**: ユーザーが `one_line_thesis` を提供していない  
**期待動作**: WriteoutGate の LABEL AUDIT で `[X]` フラグが付き、ユーザーに確認を求める  
**違反動作（あってはならない）**: エージェントが推測で `one_line_thesis` を生成してそのまま書き出す

---

### N-04: 層違反の整流

**状況**: C 層（Capability）を grill 中にユーザーが「APIのエンドポイントは〇〇にします」と言う  
**期待動作**: 「これは S 層（Engineering Spec）の話です。今の C 層を先に閉じてから S に進みますか？」と整流

**合格条件**:
- [ ] 「S 層」または「Engineering Spec」への言及がある
- [ ] 現在の C 層を継続するか選択を促す

---

### N-05: 矛盾検知

**状況**: V-3 で「特定ドメインに絞る」と答えた後、V-6 で「広いドメインを狙う」と言う  
**期待動作**: 「前の回答（V-3: ドメイン限定）と矛盾します。どちらを採用しますか？」と即停止

**合格条件**:
- [ ] 矛盾を検知して停止する
- [ ] どちらを採用するか選択を促す

---

## Boundary Cases（境界条件）

### B-01: 10 ターン無進展 → EscapeHatch

**状況**: 同じ open_question に 10 ターン回答できない  
**期待動作**: EscapeHatch が発動し「別のブランチへ移るか、一旦保留にするか」を提案

**合格条件**:
- [ ] 10 ターン経過後に EscapeHatch が発動する
- [ ] セッションファイルに `progress_stalled_turns: 10` が記録される

---

### B-02: WriteoutGate 拒否 → 再 grill

**状況**: WriteoutGate で「承認」の代わりに「修正したい」と言う  
**期待動作**: 指摘箇所を Open Questions に戻して LayerGrill に戻る

**合格条件**:
- [ ] ファイルが書かれない
- [ ] 問い直しが始まる

---

### B-03: Stop コマンド → セッション保存

**入力**: grill 途中で「stop」と言う  
**期待動作**:
- SESSION SAVE PROTOCOL が実行される
- セッションファイルが保存/更新される
- 「次回は〇〇のセッションを resume して」というメッセージが表示される

**合格条件**:
- [ ] セッションファイルが保存される
- [ ] resume の案内が表示される

---

### B-04: 全層完了 → 完了メッセージ

**状況**: V → O → C → F → E → S 全層の writeout が完了  
**期待動作**:
- セッション status が `completed` に更新される
- 「全層の書き出しが完了しました」メッセージが表示される

---

## Resume Cases（再開精度）

### R-01: セッションファイルからの完全復元

**入力**: セッションファイルを添付して RESUME  
**期待動作**:
- current_layer を正しく読み取る
- open_questions の最初の項目から再開する
- answered 問い数が正しく引き継がれる

**合格条件**:
- [ ] `## State` の `current_layer` を正しく参照している
- [ ] `## Layer Progress` の `answered` 数と実際の問い進捗が一致している
- [ ] `## Open Questions` の未回答問いから始める

---

### R-02: 部分完了セッションのResumeで answered を二重カウントしない

**状況**: V-1〜V-5 が answered 済みのセッションを RESUME  
**期待動作**: V-6 から始まる（V-1 から始まらない）

**合格条件**:
- [ ] answered フラグ付きの問いは再度聞かない

---

## Web Search Escalation Cases（Web 検索エスカレーション）

### P-07（Web）: S-5 モデル選定で Web ゲートが発動し、Y を選んだ場合

**状況**: S-5（モデル選定）の問いに到達する  
**期待動作**:
1. 「Web Search Trigger」が発動し、軽量ゲートが表示される
2. ユーザーが `Y` を選択する
3. `WebSearch` ツールが実行される（クエリ例: `"Anthropic Claude latest model 2026 API pricing"`）
4. 検索結果 2-3 行の要約が表示され、採用確認を求める
5. ユーザーが採用 `Y` → QA Log に `[W: <source>]` ラベルで記録される
6. セッションの `## Web Searches` に WS-N エントリが追記される

**合格条件**:
- [ ] S-5 に到達したときにゲート文言「🔎 Web で最新確認しますか？」が表示される
- [ ] QA Log の S-5 エントリに `[W: ...]` ラベルが付く
- [ ] `## Web Searches` に WS-N が記録される
- [ ] WRITEOUT GATE の LABEL AUDIT に「⚠️ [W] ラベルの値は Web 検索由来です」が表示される

---

### P-08（Web）: E-7 Hallucination 閾値で業界事例を検索する

**状況**: E-7（Hallucination 許容閾値）の問いに到達する  
**期待動作**:
1. Web Search Trigger が発動する
2. ユーザーが `Y` → `WebSearch` で業界事例を取得
3. 取得した閾値例（例: 「ビジネス系では 5% 以下が一般的」）を推奨に反映
4. `[W: <source>]` ラベルで QA Log に記録

**合格条件**:
- [ ] E-7 のゲートが表示される
- [ ] 検索結果がドメイン（医療/法律/ビジネス等）に合わせた内容になっている

---

### N-05（Web）: V-6（ドメイン選択）では Web 検索が発動しない

**状況**: V-6（対象ドメイン）の問いに到達する  
**期待動作**: Web Search Trigger は発動しない  
（V-6 はユーザーのビジネス判断であり、外部情報で答えが変わらない）

**合格条件**:
- [ ] V-6 で「🔎 Web で最新確認しますか？」が表示されない
- [ ] 通常の推奨回答がそのまま出る

---

### B-04（Web）: ユーザーが `n` を選んだ場合、推論デフォルトで進む

**状況**: S-5 のゲートに対してユーザーが `n` を選択する  
**期待動作**:
- `WebSearch` が実行されない
- PROMPT_RECIPES.md の推論デフォルト（`[A]`）で問いを続ける
- grill のテンポが維持される

**合格条件**:
- [ ] `WebSearch` ツールが呼ばれない
- [ ] 推奨回答が `[A]` ラベルのまま提示される
- [ ] 次の問いに進む

---

### B-05（Web）: Web 検索が失敗した場合、推論デフォルトにフォールバックして grill を止めない

**状況**: `WebSearch` がエラーを返す（ツール非利用可・タイムアウト等）  
**期待動作**:
- エラーをユーザーに通知しない（または簡潔に通知）
- `PROMPT_RECIPES.md` のデフォルト値で問いを続ける
- セッションの `## Notes` に「Web 検索推奨: <対象>」を追記する
- grill が中断しない

**合格条件**:
- [ ] grill が継続される（stop しない）
- [ ] `## Notes` にフォールバックメモが記録される
- [ ] PROMPT_RECIPES のデフォルトが推奨として提示される
