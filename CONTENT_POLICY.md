# CONTENT_POLICY.md — FREOLI Web Platform 権利ポリシー

> このポリシーは FREOLI 公式 Web プラットフォームに掲載するすべてのコンテンツ（写真・動画・楽曲・テキスト・SNS リンク）の権利取扱いを定める。
> 信頼源は本ファイル。要件定義書（`docs/requirements/freoli_web_platform_system_requirements.md` 付録 A）から派生し、運用上の最終版として扱う。

---

## 1. 基本原則

- **被写体本人の合意必須** — メンバー写真・ライブ写真は、写っている本人全員の Yes が明示的に得られたものだけを掲載する。
- **削除依頼は 1 営業日以内に対応** — 本人または関係者からの削除要請を受けたら、24 時間以内に該当コミットを `git revert` し本番反映する。
- **ニュース・コピーはえんまさのレビュー必須** — `data/news.ts` および UI コピーへの追記は、Content Owner（えんまさ＝あのむ）の事前レビューを通過したもののみ反映する。

---

## 2. メンバー写真

- 本人の Yes が出たもののみ掲載する。Yes の確認は Notion / Slack DM のいずれかに記録を残す（後述 §7 参照）。
- 削除依頼から **1 営業日以内** に削除可能な構造を維持する。`data/members.ts` から該当エントリ／写真パスを削除する PR を作り、push → Vercel 自動デプロイで本番反映する。
- **メンバー脱退時の取り扱い** は事前合意する。脱退発表後 7 日以内に本人と協議し、写真・bio・SNS リンクの扱いを決定する。

---

## 3. ライブ写真

- **撮影者の明示**（クレジット）を必須とする。撮影者不明の写真は掲載不可。
- 写っているメンバー **全員** の Yes が出たもののみ掲載する。1 人でも No があれば不掲載または該当人物のトリミングを検討する。
- **会場が撮影禁止** の場合は掲載不可。会場規約を撮影前に確認する責任は撮影者および撮影同行者が負う。

---

## 4. アー写（アーティスト写真）

- 撮影カメラマンとの **利用範囲合意** を事前に取り付ける。範囲には以下を含めて明文化する：
  - Web サイト掲載（本プラットフォーム）
  - SNS 転載（Instagram / YouTube / TikTok / X）
  - 印刷物（フライヤー・ステッカー等）
  - 第三者媒体（メディア取材記事・対バン告知等）への二次提供
- **クレジット表記** の要否を撮影者に確認し、必要な場合はサイト上およびニュース投稿で明記する。

---

## 5. 動画

- ホームページでの **動画埋め込みは現フェーズ（v0.1〜v1.0）で実施しない**。
- SNS プラットフォーム（Instagram / YouTube / TikTok）上での運用に一本化する。
- 例外として、楽曲リリース後の **Spotify / Apple Music 公式 Embed のみ可**（§6 参照）。

---

## 6. 楽曲埋め込み

- **公式配信プラットフォーム** の埋め込みのみ可とする。具体的には Spotify / Apple Music の公式 Embed コンポーネントに限る。
- リリース時期は **v0.5 以降**。v0.1〜v0.4 期間中は「2025 年配信予定」等の grayed out 表示で代替する。
- 著作権表示・配信元の明示が Embed 内で自動表示されることを掲載前に確認する。

---

## 7. ニュース投稿

- 投稿前に **えんまさのレビュー必須**。Content Owner のレビューを通過していない `data/news.ts` への追記 PR はマージ不可。
- **個人を特定する情報** は載せない：
  - 自宅住所・個人連絡先（電話番号・個人メールアドレス）
  - 第三者の実名（本人合意がない場合）
- ライブ告知では、会場の正式名称・公式 URL・公式チケットリンクを一次情報から確認した上で掲載する。

---

## 8. Yes ログ運用（被写体合意の記録）

メンバー写真・ライブ写真の掲載前に、以下のいずれかの形で本人合意を記録する：

- Notion の「写真承諾管理」ページに行追加（推奨）
- Slack DM のスクリーンショットを `docs/consent/` 配下に保存（Notion 未整備時の暫定運用）

`data/members.ts` の `consentLogged: true` フラグはこのログが存在する場合のみ立てる。

---

## 9. 削除リクエスト対応フロー

1. 削除依頼を受信（本人 / 問い合わせフォーム / SNS DM のいずれか）
2. えんまさが該当コンテンツ（写真ファイル・データ行・テキスト）を特定
3. Git で該当データ行・画像を削除する commit を作成、PR を経由せず main へ直接 push してよい（Reversible 原則により `git revert` で復旧可能）
4. Vercel 自動デプロイ → 3 分以内に本番反映
5. **削除完了を 1 営業日以内に依頼者に通知**
