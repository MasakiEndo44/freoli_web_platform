## ▼ Claude Design に貼るプロンプト本文

```
# 依頼

東京の 4 人組インディーロックバンド「FREOLI」の公式 Web サイト v0.1 のヒーロー領域・
直近ライブセクション・問い合わせフォームのデザイン案を、以下の 3 軸 × 2〜3 パターンで
合計 6〜9 案、Figma で発散してほしい。

## 役割（ヘッダーで明示）

このサイトは「新規発見の入口」ではない。SNS（Instagram / TikTok / X）で FREOLI に
出会った人物が、「ライブ参加」と「楽曲再生」という 2 つの行動に転換するための
"決断の最終確認所" として機能する。新規発見系の機能（SEO 装置・楽曲ディスカバリー）は
スコープ外。

## 成功状態（North Star Quote）

Primary Persona の自然な発話で定義された成功状態：

> 「FREOLI、Instagram で気になって HP 開いたら、次のライブの日付と会場が一瞬で
> 分かって、もう予約まで終わってた。HP の色合いも私の好きな感じだったし、
> 手よりも先に心が動かされた感じ。あの色のバンド、絶対観に行く。」
>
> ── 高橋みさき（Primary Persona、29 歳 IT 系デザイナー）

この 1 文に圧縮された 5 つの設計含意を満たす案を作る：

| 含意 | デザイン要求 |
|---|---|
| 「次のライブの日付と会場が一瞬で分かる」 | 0〜3 秒で NextLive 情報が視認可能 |
| 「もう予約まで終わってた」 | 3〜5 秒で予約導線へ到達 |
| 「色合いも私の好きな感じ」 | ネオンブルー × 黒のブランド即時伝達 |
| 「手より先に心が動かされた」 | 情動転換が情報認知より早い視覚設計 |
| 「あの色のバンド」 | "あの色" として記憶される視覚的特異性 |

## 非交渉の UX シーケンス（順序固定、変更禁止）

トップページは以下の順で縦並びの単一ページ：

  1. Hero（バンド写真 + バンド名 + 1 行キャッチ、全幅）
  2. SNS リンクバー（Instagram / YouTube / TikTok / X）
  3. 直近ライブ情報（NextLive — 今回の発散対象）
  4. メンバープロフィール（4 人）
  5. サブスク（Spotify / Apple Music、v0.1 は無効状態 "2025 年配信予定"）
  6. ニュース
  7. 問い合わせフォーム（ContactForm — 今回の発散対象）
  8. フッター（/privacy リンク）

Hero と SNS バーの上下関係を入れ替えるのも禁止。

（custom name 禁止、例：「FreoliBlue」✗）。

  Surface:          #18181b (zinc-900)
  Primary:          #22d3ee (cyan-400) / #38bdf8 (sky-400)
  Primary Hover:    #06b6d4 (cyan-500) / #0ea5e9 (sky-500)
  Text Primary:     #fafafa (zinc-50)
  Text Secondary:   #a1a1aa (zinc-400)
  Border:           #27272a (zinc-800)

タイポグラフィ：

  日本語:     Noto Sans JP (400, 500, 700)
  英文・タグ: Inter (400, 600, 800)
  見出し 1:   text-4xl / md:text-6xl, font-bold
  見出し 2:   text-2xl / md:text-4xl, font-bold
  本文:       text-base / md:text-lg
  キャプション: text-sm, text-zinc-400

スペーシング：

  セクション余白:  py-16 / md:py-24
  コンテナ最大幅: max-w-6xl, mx-auto, px-4 / md:px-8

## サウンドリファレンス（世界観の方向性）

FREOLI のサウンドは以下の系譜にある：

  - サカナクション（観察的・色彩的・余白多い）
  - BUMP OF CHICKEN（情緒的・文字大胆・物語性）
  - PEOPLE 1（ポップ・グラフィカル・若さ）
  - チョーキューメイ（オルタナ・荒さと繊細さの同居）

この 4 バンドの "となりにある音楽" として認識される視覚言語を目指す。
J-POP メジャー感・アイドル感・EDM 感・ボカロ感は避ける。

## Primary Persona

高橋みさき（29 歳・都内勤務・IT 系デザイナー、独身女性）

  - 下北沢・渋谷を生活圏。月 3〜4 回ライブハウスに足を運ぶ
  - Spotify Discover Weekly を毎週月曜朝にチェック
  - 気になったバンドはまず Instagram の固定投稿とハイライトを確認、その後 HP を見る
  - ライブ参加は「行こうかな」と 7〜8 割決まってから HP で会場確認・チケット予約
  - BUMP / サカナクション / PEOPLE 1 / チョーキューメイを日常的に聴く

痛み：HP 情報が古い／チケット予約フロー不明確／フォロー後の更新シグナル枯渇。

利用シーン：スマホで Instagram → サイト遷移 → 3 秒で次回ライブ確認 → 予約導線へ。

## 発散の評価軸（必ずこの 3 軸で展開）

### 軸 A：ヒーロー構図

  (A-a) バンド写真フル幅（4 人集合写真が画面全幅、バンド名は重ねオーバーレイ）
  (A-b) ロゴ大胆 + 写真背景小さく（"FREOLI" タイポが主役、写真は背景効果）
  (A-c) テキスト中心 + 写真サイド（キャッチコピーが左、写真が右の 2 カラム）

### 軸 B：ネオン光の表現

  (B-a) 点光源（黒地に cyan-400 がアクセントとして散る、面積比 95:5）
  (B-b) グラデーション（ヒーロー全面に sky-400 → cyan-400 のグラデーション）
  (B-c) アウトライン（テキスト・枠線・ボタンだけが光る、塗りはほぼ黒）

### 軸 C：タイポ主張

  (C-a) 日本語見出し中心（Noto Sans JP Bold で大胆に日本語キャッチを掲出）
  (C-b) 英文 "FREOLI" ロゴ中心（Inter Heavy で英文ロゴが主役、日本語は補足）
  (C-c) 混在型（英文ロゴ + 日本語サブコピーを意図的に併置）

軸 A × 軸 B × 軸 C で 3 × 3 × 3 = 27 通りあるうち、評価軸 D（下）で
「実装可逆性が高い」案に絞って、6〜9 案を発散すること。

### 評価軸 D：実装可逆性（案数には数えない、絞り込み基準）

Tailwind CSS の標準トークン（上で定義した色 / タイポ / スペーシング）だけで
再現できる案を優先。カスタム CSS や Framer Motion が必須な案は v0.1 では避ける。

## 発散の優先度（時間配分の指針）

🔴 必須：ヒーロー（主案 + バックアップ案の 2 案構成）= 各 3 案 × 2 セット = 6 案
🔴 必須：NextLive セクション（日付・会場・予約導線の階層）= 3 案
🔴 必須：ContactForm 状態デザイン（通常 / フォーカス / エラー / 成功）= 1 案を 4 状態
🟡 推奨：メンバーカードと hover インタラクション = 2 案
🟢 任意：ニュース・写真ギャラリー = 発散不要（既存プリミティブを流用する想定）

### バックアップ案（重要）

主案がアー写（プロ撮影、5/24〜25 撮影予定）前提で組まれる場合、
バックアップ案は既存ライブ写真 10 枚（プロ撮影でない、暗所・観客込み）
前提で組むこと。アー写撮影が遅延した場合のフォールバックとして機能させる。

## NextLive セクション 必須情報（要件定義書 F1.1〜F1.3）

  - 公演日（例：2025/7/11 Fri）
  - 会場名（例：Blue Sheep / 下北沢）
  - 開場・開演時刻（例：OPEN 19:00 / START 19:30）
  - チケット価格（例：¥3,000）
  - 予約導線（チケット URL 未取得のため "会場へお問い合わせください" のフォールバック UI）
  - 会場アクセス（最寄駅と所要時間：下北沢駅 徒歩 3 分）
  - Google Maps へのリンク

## ContactForm 必須項目（要件定義書 F3.6）

  - 名前（必須、1〜100 文字）
  - 連絡先メールアドレス（必須、RFC 5322 準拠）
  - 用件カテゴリ（必須、5 種から選択：出演依頼 / 取材・メディア / コラボ / ファン感想 / その他）
  - 本文（必須、10〜2000 文字）
  - Cloudflare Turnstile ウィジェット（bot 対策、フォーム下部）
  - 送信ボタン

## 状態デザイン（必須、4 状態）

ContactForm は通常・フォーカス・エラー・成功の 4 状態すべて作る。
ボタンは通常・hover・focus・disabled・loading の 5 状態すべて作る。

## 禁止表現・禁止要素（必ず遵守）

  - 楽曲未リリース期に「配信中」「リリース済み」「いつでも聴ける」等の表現を使わない
    → サブスクボタンは無効状態（grayed out）または「2025 年配信予定」テキスト
  - ボーカル歌唱動画・ライブ動画の自己ホスティング埋め込みは作らない
    → SNS プラットフォーム上で完結（v0.5 以降 Spotify / Apple Music Embed のみ可）
  - メンバー個人情報の過剰露出は避ける（本名併記は通称優先）
  - 暗すぎる配色でテキストコントラスト比 4.5:1 を下回らない（WCAG AA 準拠）

## レスポンシブ対応

以下 4 ブレイクポイントで崩れない設計：

  320px (iPhone SE 縦) / 640px (タブレット縦) / 1024px (デスクトップ標準) / 1536px (FHD)

特にスマホ（320px〜639px）を最優先（想定流入の 90% 以上がモバイル）。

## 出力形式

  - Figma ファイル 1 つ（Pages を発散案ごとに分ける）
  - 各案のキャプション 1 行（採用した軸 A / B / C を明記、例："A-a × B-a × C-c"）
  - デザイントークンは Figma Variables に上記 Tailwind 名で登録
  - レスポンシブは 320px / 640px / 1024px / 1536px の 4 サイズで Frame を用意
```

## ▼ Claude Design 使用時の運用メモ（プロンプト本文外）

### 投入後のチェックリスト

Claude Design から Figma 出力が返ってきたら、以下を確認：

- [ ] 6〜9 案が揃っているか（🔴 必須 3 種 + 🟡 推奨 1〜2 種）
- [ ] 主案 + バックアップ案の 2 案構成になっているか（アー写前提 / 既存写真前提）
- [ ] Figma の色名が Tailwind 標準名（`cyan-400` 等）で登録されているか
- [ ] ContactForm の 4 状態（通常 / フォーカス / エラー / 成功）すべて存在するか
- [ ] レスポンシブ 4 サイズ（320 / 640 / 1024 / 1536）の Frame が存在するか
- [ ] 禁止表現（「配信中」「リリース済み」等）が混入していないか
- [ ] 動画埋め込み要素が含まれていないか

### Stage 2（Day 4）のレビュー準備

選定会の前に、以下を Figma 上にコメント形式で貼っておくとレビューが早い：

- North Star Quote 5 項目評価表（各案に 5 項目 × ⚪︎/△/✗ で採点）
- 想定ユーザーヒアリングのフィードバック（Primary Persona 相当の 15 分聞き取り）
- アー写撮影スケジュール（5/24〜25）の確定 / 未確定情報

### 再発散が必要になった場合

選定会で「どの案も決め手に欠ける」となった場合、以下のいずれかを実施：

1. **評価軸の見直し**：軸 A / B / C のどれかを Persona ヒアリング結果で差し替える
2. **追加発散の限定範囲指定**：ヒーローのみ追加 3 案を Claude Design に依頼
3. **ハイブリッド検討**：既存 6〜9 案の良い要素を手動で組み合わせ、新案を 1 つ Figma 上で作成

ただし、再発散による遅延は **最大 1 日（Day 4 → Day 5 スライド）** までとし、それ以上は Phase 1 公開期限（5/31）に影響するため、主案選定を妥協してでも Stage 3 へ進む。
