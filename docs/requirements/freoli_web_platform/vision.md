# FREOLI Web Platform — Vision

## One-line Thesis

SNS で FREOLI に出会った人を、**心が先に動くネオンブルーの世界観**と、**3 秒で分かるライブ情報**で、観客とブッキング相手に変える "**決断の最終確認所**"。

短縮版（フロントマター掲出）: **「心が先に動く、ライブへの最終確認所。」**

---

## 1. Pain（解く痛み）

このプロダクトは以下 3 種の痛みを解く：

1. **音楽ファン側**: SNS（X / Instagram / TikTok）で FREOLI に遭遇した音楽好きが、「気になるバンドが見つかったが、次のライブはいつ・どこで・どう予約するかが分からない」「楽曲も無いので何者か掴めない」── 興味から行動転換するための **最後の一押し情報が点在している**。
2. **業界関係者側**: ライブハウスのブッカー・メディア関係者・対バン候補バンドの **ブッキング・取材・コラボ依頼の正規窓口が SNS DM に分散**し、機会が取りこぼされる。
3. **バンド側**: 楽曲未リリース期に「このバンドはどんなバンドか」を **集約して伝える単一の窓口がない**。

> 引用元: `docs/requirements/freoli_web_platform_detailed_requirements_specification.md` §2「対象課題」

---

## 2. 3 年後の差分

**ある場合（2029 年）**: SNS → サイト → 行動転換という装置を通じて、独自ドメイン（v1.0 で取得）下に **ライブ履歴 / 楽曲 / メディア掲載 / 業界連絡先** が時系列で蓄積された "**信頼の資産**" が形成されている。SNS で初遭遇したリスナーは "本物っぽさ" を 90 秒で確認でき、ブッカーは 5 分でブッキング判断ができる。

**ない場合**: FREOLI の存在証明が SNS の流れる投稿に依存し続け、「決断の最終確認」を求めるリスナーが Lit.Link 系の汎用ページや古い情報で離脱する。ブッキング機会が DM の海に沈み、メンバー個人へ問い合わせが分散して取りこぼされる。サーキット出演交渉で「公式サイトが無いバンド」として土俵に上がれない局面が発生する。

**差分の核**: 「FREOLI を本気で観察する人（リスナー上位層・業界）」の **信頼形成と行動転換の歩留まり** が決定的に変わる。SNS だけでは作れない "落ち着いて読める一次資料" を保有しているかどうか。

---

## 3. 我々がやる必然性

汎用ツール（Lit.Link / linkpop / 他バンドの汎用 HP）ではなく FREOLI 自前のプラットフォームでやる必然性は、4 つの「他に置けない資源」に依存する：

1. **Cornered Resource（独占資源 = ゆうすけの声）**: 業界人（ゆうたさん）も「歌で何を伝えるかが核」と評価。Lit.Link 系の汎用ページでは、この非対称性をヒーローに据えられない。FREOLI が自前で世界観・ヴィジュアル・声の入口を設計できる場所が必要。
2. **Counter-Positioning（カウンターポジショニング）**: インディーバンドの大半が Lit.Link 系に依存する中で、ネオンブルー × 黒の独自世界観と「ライブ集客に特化した行動転換型 UI」自体が差別化。"リンクを並べるだけ" を超える設計思想を持つこと自体が moat になる。
3. **Branding**: サカナクション譲りの観察的・色彩的トーンを長期で蓄積するには、汎用テンプレートでは不可能。`v0.1 → v0.5 → v1.0` のロードマップで節目ごとに進化させる "**成長する一次資料**" として、自前運営が必須。
4. **運用面（権利ガバナンス）**: メンバー写真・ライブ写真の **被写体本人の Yes 必須、削除依頼 1 営業日以内** という運用は、汎用プラットフォームでは保証できない（`AGENTS.md` §6 Layer 2 / `CONTENT_POLICY.md`）。

---

## 4. Non-goals（絶対に目指さない方向）

1. **「新規発見の入口」になること**: SEO 集客装置・楽曲ディスカバリーサイト化は目指さない。新規発見は SNS（Instagram / TikTok / X / YouTube）が担い、本サイトは "決断の最終確認所" に徹する（`AGENTS.md` 設計原則 ①）。
2. **動画の自己ホスティング**: ボーカル歌唱動画・ライブ動画をホームページ内に埋め込まない。SNS プラットフォーム内で完結。Spotify / Apple Music の公式 Embed のみ可（v0.5 以降）。
3. **動的バックエンド・DB の保有**: ユーザー認証・ファンクラブ会員・コメント・EC は v1.0 までスコープ外。Next.js + 静的データファイル（`data/*.ts`）+ Resend だけで完結する構成を維持し、Vercel revert で安全に戻せる Reversibility-First を保つ（`AGENTS.md` 設計原則 ②）。
4. **被写体本人未承諾コンテンツの掲載**: メンバー写真・ライブ写真は本人の Yes が出たもののみ。"とりあえず載せて後で承認" は禁止（`AGENTS.md` 設計原則 ③）。
5. **誤認誘導表現**: 楽曲未リリース期に「配信中」「リリース済み」等の表現を使わない。「2025 年配信予定」または無効状態で表示する。
6. **独自ドメインの早期取得**: v0.5 楽曲リリース時には取得せず、**サーキット出演確定時（v1.0）** まで `*.vercel.app` で運用（設計決定 #8）。

## 6. 対象ドメイン

- **地理**: 東京・関東圏（下北沢・渋谷・新宿エリアのライブハウス文化圏）
- **音楽ジャンル**: 日本のインディーロック／オルタナティブロック。サカナクション・BUMP OF CHICKEN・PEOPLE 1・チョーキューメイ系統を聴く層が "となりにある音楽" として認識する領域。
- **業界ポジション**: ライブハウス回路の "次のサーキット出演候補" 帯。100 人動員 → サーキット出演 → 全国規模認知 へのロードマップ上の **「無名 → 中堅」境界域**。
- **時間軸**:
  - **v0.1（〜 2025/5/31）**: 7/11 Blue Sheep ライブ集客（30〜40 人動員）が最初の対象イベント
  - **v0.5（楽曲リリース時）**: 都内ライブ通常回路で 60〜80 人動員、Spotify / Apple Music 初回再生獲得
  - **v1.0（サーキット出演確定時）**: 100 人動員達成、独自ドメインで全国向け情報発信

**対象外**: 海外市場、メジャーレーベル契約バンド層、アイドル・ボカロ・EDM 等の隣接ジャンル。

## 7. North Star Quote

成功した状態を、Primary Persona の自然な発話で定義する：

> 「FREOLI、Instagram で気になって HP 開いたら、次のライブの日付と会場が一瞬で分かって、もう予約まで終わってた。HP の色合いも私の好きな感じだったし、**手よりも先に心が動かされた感じ**。あの色のバンド、絶対観に行く。」
>
> ── 高橋みさき（Primary Persona）の想定発話

この 1 文には 5 つの設計含意が圧縮されている：

| フレーズ | 設計含意 |
|---|---|
| 「次のライブの日付と会場が一瞬で分かる」 | F1（ライブ集客機能）— 3 秒で把握 |
| 「もう予約まで終わってた」 | チケット導線の摩擦最小化 |
| 「色合いも私の好きな感じ」 | ネオンブルー × 黒のブランド即時伝達（F2.1 ヒーロー） |
| 「手よりも先に心が動かされた」 | **認知より早い情動転換**。決断の最終確認所として、迷いを溶かす速度 |
| 「あの色のバンド」 | Branding 資産化（"あの色" として記憶される） |

## 8. このビジョンと矛盾する設計判断が出たときの照合ルール

- **「決断の最終確認所」を逸脱する機能提案は却下**。新規発見・SEO 集客装置化に向かう提案が出たら、本ファイル §4 Non-goals に照合する。
- **AGENTS.md と本ファイルが矛盾した場合は本ファイル（要件定義書系）が優先**（user memory: `feedback_requirements_priority.md`）。本ファイルの更新時は AGENTS.md も追従させる。
- **「手よりも先に心が動かされた」を侵食する判断は却下**。情報設計が認知より遅くなる UI（モーダル待ち、スプラッシュ、過剰なローディング等）は本ビジョンに反する。
