# FEAT-003: Members（メンバープロフィール）

## F-1 Persona

- **Primary**: 高橋みさき（「どんな人たちのバンド？」を確認する 30 秒）
- **Secondary**: 田村ブッカー（メンバー構成・パートを 5 分以内判断の一部に）

## F-2 Trigger

ヒーロー → ライブ情報を見終わって、メンバーセクションまでスクロールした瞬間。

## F-3 Done When

訪問者が **4 人のメンバー名（通称）・パート・顔・個性** を把握し、「このメンバー、なんか良さそう」と親近感を持った状態。業界関係者にとっては **構成と楽器配置が即時判明**。

## F-4 上位 Capability

- **メイン**: CAP-002 世界観・正体の即時伝達
- **連携**: CAP-004 権利ガバナンス（写真承諾の制約）

## F-5 User Story

> As 高橋みさき, I want **4 人のメンバー名（ゆうすけ / あのむ / ひろむ / Aberyo）・パート（Vo.Gt / Gt / Ba / Dr）・顔写真・1〜2 行の個別紹介を一覧で見られたら**, so that **「この 4 人の音、想像できる」と感じてバンドへの親近感が芽生える**。

## F-6 受け入れ基準（概要、E 層で EARS 化）

1. 4 人分のカードが **横並び（デスクトップ）/ 縦並び（モバイル）** で表示
2. 表記順は **楽器順（Vo/Gt → Gt → Ba → Dr）**（detailed §4.2 と整合）。通称・パート・顔写真・1〜2 行紹介を含む
3. **被写体本人の Yes が取れた写真のみ表示**（CAP-004 制約）。未承諾は **パートを示すシルエットアイコンで代替**

## F-7 Out of Scope

- メンバー個別の詳細ページ（v1.0 プレスキット時に検討）
- メンバー個人 SNS リンク（プライバシー観点でスコープ外。バンド公式 SNS のみ CAP-007 で扱う）
- メンバー個人 YouTube チャンネル（v3 §追加情報リクエスト 🟢 低優先度 #3、現フェーズ未確定）
- メンバーインタビュー記事

## F-8 実装フェーズ

**Phase 1（v0.1、2025/5/31）必須**。Week 2 中盤（5/26〜28）に実装。写真承諾は 5/29〜31 で取得。
