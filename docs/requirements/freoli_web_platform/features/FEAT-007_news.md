# FEAT-007: News（お知らせ／ニュース欄）

## F-1 Persona

- **Primary**: 高橋みさき（バンドが「活動中」であることのシグナル確認、再訪動機）
- **Secondary**: 田村ブッカー（最近の活動・メディア露出を 1 分で把握）

## F-2 Trigger

サブスクリンクまでスクロールし「最近どうしてるんだろ？」と思った瞬間。あるいは SNS で「メディア出演しました」投稿を見て詳細を確認しに来た瞬間。

## F-3 Done When

訪問者が **最新のライブ告知・楽曲リリース・メディア出演を時系列で把握**し、「このバンド活動中だな」「定期的に更新されている」と認識した状態。再訪意欲が形成される。

## F-4 上位 Capability

- **メイン**: CAP-008 ファン継続接点の維持
- **連携**: CAP-004（投稿前レビュー制約）

## F-5 User Story

> As 高橋みさき, I want **ニュース欄で最新 3〜5 件の活動告知（ライブ・楽曲・メディア）が時系列降順で見られたら**, so that **「活動中・更新されている」と確認して、フォロー継続 or 再訪する動機が生まれる**。

## F-6 受け入れ基準（概要、E 層で EARS 化）

1. **時系列降順で 3〜5 件表示**（日付・タイトル・本文・タグ）
2. v0.1 では `data/news.ts` 直書きで管理、Git PR レビューを **えんまさが必ず実施**（F4.2、CAP-004 連携）
3. **月 1 回以上の更新を運用目標**（カレンダーリマインダー固定: 毎月 1 日。Content Owner = えんまさ）

## F-7 Out of Scope

- コメント・「いいね」機能（v1.0 までスコープ外、AGENTS.md 設計原則 ② DB 保有禁止）
- RSS / Atom フィード（v0.5 以降検討、ペルソナの主要購読チャネルは SNS）
- ニュース個別詳細ページ（v0.1 では一覧表示のみ、長文は本文に直書き）
- CMS 連携（v0.5+ で microCMS / Notion API 検討、F4.3）
- ニュース通知メール配信（v1.0 メーリングリストで検討、🟢 低優先度 #1）

## F-8 実装フェーズ

**Phase 1（v0.1、2025/5/31）必須**。初回投稿: **7/11 Blue Sheep ライブ告知**（v3 中優先度 #3、文面確定要）。

## 既存要件対応

- F4.1（ニュース欄、必須）
- F4.2（レビュー手順、必須）
- F4.3（CMS、任意・中長期）
- 設計決定 #15（ニュース投稿レビュー必須）
